## What breaks the cache

These silently invalidate — no error, no warning:

- **Edit CLAUDE.md** — injected into the system prompt; even one space breaks it
- **Change hooks or permissions** — both are embedded in the system prompt
- **Add MCP server or /skill** — changes tool definitions, which are first in the token stream
- **/reload-plugins** — rebuilds the tool list from scratch
- **Switch model** — cache is model-scoped; different model = cold start

Safe actions that **don't** break the cache: normal turns, subagent calls, reading/editing files. /compact keeps tools and system prompt cached but rewrites history, so prior turns are reprocessed.

<CacheCallout>
The KV cache saves work within one request. The prompt cache saves the same
K,V vectors across requests — so the next call with the same prefix skips
prefill. Keep stable content at the top, don't touch it mid-session, and
the cache stays warm.
</CacheCallout>
