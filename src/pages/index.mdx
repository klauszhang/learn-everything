## Every turn re-reads everything

The model has no memory between requests. Each turn sends the full conversation
from scratch — system prompt, tools, files, history, then your new message.
Hover a turn to see the repeated prefix:

## What breaks the cache

The cache requires an exact token-by-token prefix match. Click each scenario
to see which segments go cold:

<CacheInvalidationDemo client:load />

<CacheCallout>
The KV cache saves work within one request. The prompt cache extends the same
idea across requests — if the prefix matches, the model skips recomputing it.
Keep stable content at the top, don't touch it mid-session, and the cache
stays warm.
</CacheCallout>
